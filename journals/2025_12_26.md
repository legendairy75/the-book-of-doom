## Understanding Errors

```
- Erorrs in Unix ( unlike in other os) *usualy* tell you the problem
- ### Common errors
- Permission denied
	- you are trying to read, write or exicute a file but you dont have permision to
- Operation not permited
	- you are trying to kill a process you dont own
- Segmintal error
	- the program is trying to access a part of memory it is not alowed to tutch
```

- \[\[Listing and manipulating processes\]\]
  - Each process has a numeric *process ID (PID)*
  - to list pid type `ps`
  - Field

    | Desc |
    |------|

    |      | PIDthe process id                                     |
    |------|-------------------------------------------------------|
    | TTY  | terminal where process is running                     |
    | STAT | process status(S means sleeping while R means running |
    | TIME | The amount of CPU time The process has used           |
    | CMD  | The command used to run the program                   |
- Process termination
  - To terminate a process you send it a *signal* with the kill command
  - `kill *pid*`
  - to freez a process `kill -STOP *pid*`
  - use `kill -CONT *pid*` to continue running a process
  - use *CTRL-C* to kill process running in the current terminal
  - If allelse fails use `kill -KILL`
    - be cairful with this one
  - you can stop a job in the current terminal with *CTRL-Z*
    - after type `fg` to bring job to forground or `bg` to put job in background
  - To start a job in the background us *&*
    - `gunzip file.gz &`
