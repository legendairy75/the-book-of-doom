## redirecting standard in/out
	- you can redirect the output of one command into the input of another with [*>*](((69498726-8df6-4b16-b7c6-753a469f84dc)))
	- `ls --help > lstext` will print the output of the ls help page to a file called 'lstext'
	- `>>` appends to the end of a document
	- to sent stdout of oner command to stdin of another use `|`
- ## standard error
- you can redirect standard error to a file with `2>`
	- `ls /fffffffff > f 2>e` will print the errir into *e*
	- the *2* specifies the *stream ID*. *1* is stdout id
- you can also redirect stderr to the same file as stdout with `2>&1`
- ## standard input redirect
	- to channel a file to a programs stdin use `<` operator
	- most programs accept files without needing to use *<* but some may
